# 定稿卡片与适配模型收口设计

## 目标

本次改动同时解决两类问题：

1. 定稿池交互混乱
2. 适配版本生成链路不可信

用户已经确认新的产品目标：

- 候选图每点一次“选定稿”，只生成一张独立的定稿卡片
- 一张定稿卡片只绑定一张原始定稿图，不再把多张原图混放在同一个定稿池卡片里做适配
- 卡片内必须先选择渠道，选完渠道后才动态显示该渠道对应的版位
- 版位分为“可直接导出”“需适配”“暂不支持”
- 适配资产按“定稿卡片 + 比例”存储，不按渠道重复生成
- 同一比例只保留一张当前生效图，不满意时通过“重新生成”直接覆盖
- 适配版本生成和重生成都必须把当前卡片的原始定稿图作为参考图传给模型
- 适配模型下拉只保留已验证支持参考图链路且支持 `1:1 / 3:2 / 16:9 / 9:16` 四个比例的模型
- `Gemini Flash` 不再出现在适配模型里

## 背景问题

当前实现存在四类结构性问题：

1. 定稿池以“分组集合”思维承载多张原图、多渠道、多比例，用户难以判断某张适配图到底从哪张原图派生
2. 渠道和比例混在同一层，用户在未选择渠道前就会面对一整片版位和操作入口，信息过载
3. 适配模型列表没有经过严格收口，用户能选到实际上无法稳定完成“参考图 + 目标比例适配”的模型
4. 参考图与提示词快照链路不可信，至少已暴露出：
   - 即梦适配时疑似未正确传参考图
   - 通义千问适配时未稳定产出目标比例
   - 查看提示词时会出现“无效的参考图地址”，无法证明真实请求内容

## 方案对比

### 方案 A：独立定稿卡片 + 单渠道展开 + 比例资产复用

每张被选中的候选图生成一张独立定稿卡片。卡片内部先选渠道，再按该渠道显示版位状态；适配资产按比例存储并在不同渠道间复用。

优点：

- 原图与派生图关系最清楚
- 用户不用再思考多张原图之间的交叉复用
- 渠道只负责“当前需要哪些版位”，不再负责持有图片资产
- 同比例适配图可跨渠道复用，减少重复生成

缺点：

- 定稿图较多时，定稿卡片数量会增加

### 方案 B：独立定稿卡片 + 渠道内各自保存适配图

也是每张原图一张卡片，但 `OPPO 16:9`、`VIVO 16:9` 分别保存。

优点：

- 语义直白

缺点：

- 同比例资源重复生成
- 导致用户在不同渠道间看到重复资产
- 不符合“比例相同即可复用”的已确认需求

### 方案 C：公共定稿池 + 全局版位工作台

所有原图集中到一个池子里，再给不同渠道和版位做统一匹配。

优点：

- 集中管理

缺点：

- 会回到当前产品已经暴露出来的混乱状态
- 仍然难以回答“这个版位到底基于哪张原图”

## 方案结论

采用方案 A。

核心产品模型收口为：

- 一张原始候选图
- 对应一张独立定稿卡片
- 卡片内部持有一张原始图和若干按比例存储的派生资产
- 渠道只负责映射版位需求
- 导出按“当前渠道的版位”执行，不按渠道保存额外图片

## 交互设计

### 1. 候选图选定稿

候选图池中的每一张候选图保留“选定稿”操作。

点击后：

- 若该候选图尚未定稿，则创建一张新的定稿卡片
- 若该候选图已经对应一张定稿卡片，则不再重复创建

### 2. 定稿卡片结构

每张定稿卡片只展示和处理一张原始图：

- 原始定稿缩略图
- 原始比例
- 来源信息
- 查看提示词
- 取消定稿
- 渠道选择区
- 渠道展开后的版位区
- 已有比例资产区
- 导出区

### 3. 渠道选择

渠道是第一层入口。未选渠道前，不显示版位区。

交互规则：

- 点击 `OPPO` 后，仅展开 `OPPO` 的版位面板
- 点击 `VIVO` 后，仅展开 `VIVO` 的版位面板
- 不做多渠道并行展开
- 渠道切换时只切换当前视图，不复制资产，不触发自动生成

### 4. 版位分区

选定渠道后，该渠道的版位按以下三组展示：

1. `可直接导出`
2. `需适配`
3. `暂不支持`

其中：

- `可直接导出` 默认全部不勾选，用户自行勾选要导出的版位
- `需适配` 允许用户勾选后点击“生成适配版本”
- `暂不支持` 仅展示说明，不提供操作入口

### 5. 比例资产与版位关系

比例资产属于“定稿卡片”，不属于“渠道”。

例如，一张卡片当前已经有：

- 原图 `1:1`
- 派生图 `16:9`

则：

- `OPPO` 需要 `16:9` 时直接复用该派生图
- `VIVO` 需要 `16:9` 时也直接复用该派生图

渠道差异只保留在导出阶段处理，例如：

- 导出尺寸
- 压缩质量
- 文件命名

### 6. 同比例只保留一张当前图

一张定稿卡片内，同一个比例只保留一张当前生效图。

若用户对该比例结果不满意：

- 点击“重新生成”
- 系统再次以原始定稿图为参考图生成同一比例
- 生成成功后直接覆盖当前比例图

不保留同一比例的多版本候选。

## 状态流转

### 初始态

- 候选图未定稿

### 选定稿后

- 创建独立定稿卡片
- 卡片只包含原始图
- 还未选择渠道，版位区隐藏

### 选择渠道后

系统读取该渠道固定版位定义，并逐个判定：

- 卡片里已有对应比例资产 -> `可直接导出`
- 卡片里没有对应比例资产 -> `需适配`
- 当前产品明确不支持 -> `暂不支持`

### 生成适配版本

用户在 `需适配` 中勾选若干项后点击生成。

系统先将版位需求收敛为“目标比例集合”：

- 同一渠道内多个版位要求相同比例时，只生成一次该比例
- 例如：两个版位都需要 `16:9`，只生成一张 `16:9`

生成成功后：

- 新比例资产写回当前卡片
- 所有依赖该比例的当前渠道版位自动转入 `可直接导出`

### 切换渠道

- 不新建卡片
- 不复制比例资产
- 仅重新映射当前渠道的版位状态

### 重新生成

- 对已有比例资产再次生成同样比例
- 仍然使用原始定稿图做参考图
- 成功后覆盖旧图

### 删除派生图

- 允许删除派生比例图
- 不允许删除原始母图
- 删除后，该比例从 `可直接导出` 回退为 `需适配`

## 适配模型设计

### 模型列表收口

“适配模型”不再直接复用所有生图模型，而是收口为单独的白名单。

只有同时满足以下条件的模型，才允许出现在适配模型下拉中：

1. 已验证支持参考图链路
2. 已验证支持 `1:1 / 3:2 / 16:9 / 9:16`
3. 已验证能把输出稳定生成到目标比例
4. 已验证当前网关请求格式和本项目调用格式相匹配

不满足任一条件的模型：

- 不显示在适配模型下拉里

### Gemini Flash

`Gemini Flash` 直接从适配模型列表移除，不再参与本功能。

### 其他模型

即梦、通义千问以及其他可能保留的模型，需要逐个验证：

- 参考图请求是否真实生效
- 比例参数格式是否正确
- 返回结果是否真的是目标比例

若无法稳定覆盖四个目标比例，则从适配模型列表删除。

## 参考图与快照规则

### 适配生成

适配版本生成时，必须使用“当前定稿卡片的原始定稿图”作为参考图。

禁止：

- 不传参考图
- 仅在 UI 上声称用了参考图，但真实请求没传
- 使用其他派生图作为参考图继续套娃
- 使用组级回退字段伪造参考图状态

### 重生成

重生成仍然必须使用原始定稿图作为参考图，不允许拿上一次适配结果继续当参考图。

### 查看提示词

查看提示词必须展示单图级真实请求快照，至少包含：

- 正向提示词
- negative prompt
- 模型
- 目标比例
- 真实参考图列表

“参考图”必须来自该次真实请求快照，不再允许显示“无效的参考图地址”之类无法证明真实请求内容的占位状态。

若历史图片缺少快照，则明确提示：

- 该图片缺少历史生图快照，请重新生成后查看

## 生成链路验收要求

本次必须把以下问题纳入同一批修复：

### 1. 即梦参考图链路

至少核对并修复即梦适配时的参考图传递，确保：

- 真实请求里包含参考图
- 模型拿到的参考图就是当前卡片的原始定稿图

### 2. 通义千问比例适配链路

至少核对并修复通义千问适配时的目标比例链路，确保：

- 例如 `9:16 -> 16:9`
- 结果图必须实际生成成 `16:9`
- 不是只在前端文案里显示目标比例

### 3. 适配模型白名单

最终留下的适配模型必须都能稳定支持：

- 参考图
- 四个标准比例
- 当前项目的请求格式

### 4. 提示词快照

查看提示词展示的参考图、模型和比例，必须能够回溯到单图级真实请求快照。

## 空态与异常

### 未选渠道

显示提示：

- 请选择渠道后查看该渠道版位

### 全部已就绪

- 只展示 `可直接导出`
- 提示当前渠道所需比例已齐备

### 全部需适配

- 只展示 `需适配`

### 同比例多版位

`需适配` 中按比例去重显示，例如：

- `16:9（用于：横版信息流、横幅大图）`

### 生成失败

- 该比例保持失败状态
- 不进入 `可直接导出`
- 保留 `重新生成`
- 错误信息尽量具体到模型、参考图、比例参数或网关异常

### 暂不支持

- 在当前渠道版位区单独列出
- 不可勾选
- 不可生成

## 实现影响面

预计会影响以下模块：

- 候选图“选定稿”后的定稿节点构建方式
- 定稿卡片数据结构
- 定稿卡片 UI
- 渠道版位判定与导出逻辑
- 适配版本生成 API
- 图像生成服务的参考图路由和模型分流
- 单图级请求快照结构
- 适配模型配置来源

## 验收标准

### 交互验收

1. 候选图每次“选定稿”只生成一张独立定稿卡片
2. 一张定稿卡片只绑定一张原始图
3. 未选渠道前不展示版位区
4. 选择某渠道后，只展示该渠道的版位
5. `可直接导出` 默认不勾选
6. 同比例派生图可跨渠道复用
7. 同一比例重生成后直接覆盖旧图

### 生成验收

1. 适配版本和重生成都使用原始定稿图作为参考图
2. 即梦链路真实传入参考图
3. 通义千问链路真实产出目标比例
4. 不满足四比例能力的模型不会出现在适配模型列表
5. `Gemini Flash` 不会出现在适配模型列表

### 快照验收

1. 查看提示词展示真实单图级请求快照
2. 参考图展示的是真实请求快照里的地址
3. 缺失历史快照时明确提示用户重新生成

## 不纳入本次

- 同一比例保留多版本供用户挑选
- 多张定稿卡片的一键批量导出
- 按渠道分别保存同比例派生图
- 自动补录历史图片的请求快照
